Froozhie Information

FroozHIE Tool

Successful Health Information Exchange (HIE) requires electronic sharing of patient health information. When a physician receives a medical summary from an outside source (hospital, referring physician, patient, etc.), the data must be viewed, compared and reconciled with existing data in the practice EMR/EHR. This is usually accomplished with a time-consuming, labor-intensive and error-prone manual process. The goal of the FroozHIE is to allow for easy, customized viewing of CDA files, and ultimately easy reconciliation between two sources of information provided via CDA (usually an incoming file, and existing information within a system).

Features/Instructions

  • Opening a file: To open a new CDA file, click the "Add File" file icon in the upper left of the page. Once the "Add File" modal window opens, click on the "Choose File" button to open the file selection window, and find your desired file to open it within the tool.
  • Viewing a file: Once a file is opened, it will be viewable within a tabbed element on the page. A standard stylesheet is used for display (for security and usability reasons, outside stylesheets designated within the CDA file are not used in the FroozHIE tool.) All appropriate data is displayed, other than what has been previously designated to be hidden (described in "Settings" below). If you would like to view the raw XML of the file, a "View XML" button is available in the upper left. You may close a file tab by clicking the "x" in the tab, or you can close all files by clicking the "Close All" button in the upper right of the page.
  • Viewing Raw XML: When viewing raw XML, there are three buttons available to help in your viewing. The "collapse all" and "expand all" buttons will collapse or expand all of the nodes in the file. The "comments on/off" button will toggle display of the comment elements within the file. Also of note, multiple instances of a node type at the same node level will be numbered past the first instance (the multiple "Component" nodes under "structuredBody" will show a good example of this feature).
  • Settings: Under settings, you can set whether the CDA Header is in "Minimalist" mode. When this is turned on, only a few demographic sections in the header are display on load. A button to expand to show the full header will be displayed when this mode is enabled. The order of the sections within the CDA can also be set on the settings page. Sections are only added to the list once you have opened a file, and the sections within that file are added to the sections list for ordering. To change the order, drag the section box to the desired position. If you wish for the section to be completely hidden on load, move the section to the "Removed sections" list at the bottom of the settings window. The "clear" button allows you to clear the list of sections and start over with ordering.

Future planned enhancements/features:

  • Order by document type: Section ordering/visibility settings to be set by CCD document type (Summary of Episode Note, Care Plan, etc.), so you may display different document types their own way.
  • File Merging: Ability to take two files, view differences between the files, and merge the sections into one resulting XML file.
  • Application integration: Settings that will allow for easier integration into other applications (ability to load a single designated file on page load, etc).
